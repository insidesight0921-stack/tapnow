# Tapnow 플랫폼 개발 컨텍스트 (2026-04-01 기준)

이 문서는 다음 대화 세션에서 작업을 원활하게 이어가기 위해 현재까지의 구현 상태와 주요 아키텍처 결정을 요약합니다.

## 1. 프로젝트 개요
- **목표**: 주짓수 도장 관리 SaaS 플랫폼 (`tapnow`) 구현.
- **기술 스택**: React, Vite, TypeScript, Zustand (상태 관리), Lucide-react (아이콘), CSS.
- **인증 시스템**: 
    - `LoginPage`: '로그인' 및 '회원가입' 기능 포함.
    - 역할 기반 접근: `SUPER_ADMIN` (admin@tapnow.com), `GYM_ADMIN` (개별 도장 관리자).

## 2. 주요 구현 현황
### SaaS 구독 시스템
- **요금제**: `free` (30명 제한), `basic` (6,900원), `plus` (14,900원, 카카오톡 포함).
- **제약 로직**: `free` 요금제 도장이 30명 초과 시 경고 배너, 35명 초과 시 추가 등록 차단.

### 데이터 격리 (Multi-tenancy)
- 모든 데이터(`Members`, `Attendance`, `Payments`)는 `gymId` 필드를 가짐.
- 로그인한 도장 관리자는 본인의 `gymId`에 해당하는 데이터만 조회/수정 가능.

### UI/UX 고도화 (최근 완료)
- **지표 재정의**: '미납'을 '만료' 개념에 통합하고 사이드바 배지 제거.
- **출석 관리**: '장기 미출석' 기준을 **14일(2주)**로 단축.
- **회원 등록 폼**: 이름, 전화번호, 등록/시작일, 요금제를 필수값으로 설정하고 요금제 선택 시 **금액 자동 입력** 구현.
- **레이아웃**: 프로필 이미지를 도장 이름 옆으로 이동, 헤더 버튼 명칭(`백업하기`, `불러오기`) 및 순서 최적화.
- **모바일 최적화**: 375x812 해상도에서 모든 관리자 페이지 레이아웃 정합성 확보.

## 3. 테스트 계정 정보
- **슈퍼 관리자**: `admin@tapnow.com` / `admin`
- **테스트 도장**: `gym@tapnow.com` / `abc1234`

## 4. 향후 작업 과제 (Roadmap)
1. **결제 연동**: 실제 PG사 연동을 통한 요금제 업그레이드 기능.
2. **카카오톡 알림**: PLUS 요금제 사용자를 위한 알림톡 API 연동.
3. **데이터 영속성**: 현재 Zustand 메모리 기반 데이터를 **Firebase Firestore**로 마이그레이션.
4. **키오스크 고도화**: 출석 시 남은 횟수/기간 표시 및 만료 예정 안내 강화.
