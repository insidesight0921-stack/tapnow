# AI Assistant Verification Policy

이 문서는 AI 어시스턴트(봇)가 프로젝트 코드를 수정하고 검증할 때 반드시 지켜야 할 체크리스트 및 원칙을 정리한 문서입니다. AI는 코드 수정 요청을 받을 때마다 다음 원칙을 준수해야 합니다.

## 검증 프로세스 규칙 (SOP)

1. **로컬 검증과 클라우드 검증의 분리**
   - 개발 환경(로컬)에서의 정상 동작 확인으로 모든 검증이 끝났다고 판단하지 않는다.
   - Vercel과 같은 실 배포 서버(URL)가 존재하는 경우, **반드시 재배포(Git Push) 후 실제 URL에 접속하여 최종 확인(Visual QA/End-to-end Test)**해야 한다.

2. **재배포 및 대기 절차**
   - 코드를 수정 및 커밋한 후 `git push`를 실행한다.
   - Vercel 등 배포 플랫폼의 빌드 파이프라인이 완료되기 전까지(통상 1~2분 소요) 브라우저 검증을 성급하게 진행하지 않는다. 빌드 타임 동안 충분히 대기한다.

3. **검증 방식**
   - 고객의 브라우저 세션(ex. 현재 띄워놓은 구글 크롬)은 봇이 직접 조종할 수 없으므로, 봇 전용 브라우저 에이전트(Playwright/Puppeteer 등)를 활용해 독립적으로 URL에 접속하여 기능과 UI를 직접 클릭하고 스크롤하여 검증한다.
   - 데스크톱 화면뿐만 아니라 모바일(375px), 태블릿 뷰포트에서의 UI 깨짐이나 Text Overflow(말줄임표 등) 증상을 철저히 점검한다.

4. **프로덕션 코드 청결도 유지**
   - 개발자가 디버깅을 편하게 할 목적으로 임시 스크립트(숨김 버튼, 콘솔 삽입용 자바스크립트 스니펫 등)나 불필요한 단축키 리스너를 실 서비스 코드(App.tsx 등 메인 진입점)에 삽입하지 않는다.
   - 코드는 오직 고객이 요청한 비즈니스 로직과 UI 개선에만 집중해야 하며, 사이드 이펙트나 지저분한 코드가 프로덕션에 올라가지 않도록 철저히 클린 코드를 작성한다.
